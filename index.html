Prework Study Guide

HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • HTML serves to get content framework on the page.
  • Easy to use, easier to re-use.
  • It serves well to give an overall idea of the page content.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • It's always good to reuse code and not duplicate your work, DRY.
  • CSS will give the specification for the content in our HTML file.
  • You can add color, size, and organize the page content in style.

GIT

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git pull origin main: pulls last repository update.
  • git commit -m "Example": will make the commit after using the git add -A feature.
  • Once you commit your changes you can push the branch update into the main branch to review and merge.

JAVASCRIPT

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • "Normal" Coding, in easy words is just the way to manage content rules to interact with the file.
  • Rules can be as easy as a comparisson as complex to use data analysis in static repositories where data go in n' out.
  • Programming skills will define the complexity of the JS file.
